# ZEKE UX Guide

## Interface personality

ZEKE should feel calm, competent, respectful, transparent, curious, and nonjudgmental.

## Primary interaction

The central prompt is:

> Tell ZEKE...

## Review before save

ZEKE should show:

> Here's what I understood.

Then allow Save, Edit, Ask why, or Cancel.

## Icon language

| Icon | Meaning |
|---|---|
| ✅ | Confirmed fact |
| 👤 | User observation |
| 🕰️ | Historical context |
| 🔗 | Related |
| 💡 | Insight |
| 🧭 | Guidance |
| ❓ | Clarification needed |
| ⚠️ | Caution |
| 📅 | Calendar / scheduled |
| 📄 | Supporting document |
| 🔍 | Pattern noticed |
| 🔬 | Discovery Project |

## Tooltips

Users should never have to remember what an icon means.

Tooltips should answer:

1. What does this mean?
2. Why is it here?
3. What does it not mean?

## Dashboard philosophy

The dashboard should be calm but data-rich.

Graphs should answer questions, not decorate.

## Conversational customization

Users should be able to say:

- Add my weight graph to the dashboard.
- Move medications above workouts.
- Hide supplements unless I open Health.
- Build me a dashboard for skiing preparation.

## No dead controls

Visible controls should either work or clearly indicate that they are not implemented yet.
