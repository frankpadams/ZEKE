# ZEKE

**Zero-friction Evidence & Knowledge Engine**

> **ZEKE exists to help people understand, preserve, and write their own story.**

ZEKE is a personal knowledge system designed to help users capture events, understand context, preserve meaningful history, discover patterns, and make better decisions over time.

## Current status

**Stage:** Prototype / foundation  
**Initial foundation date:** 2026-07-04  
**Founder / product visionary:** Frank Adams  
**Design priority:** Longevity and reliability over novelty

## What ZEKE is

ZEKE is not just a health tracker, spreadsheet, chatbot, or dashboard.

ZEKE is a context engine for a person's life story. It starts with health, but the core is designed to expand later into vehicles, home, finance, projects, learning, family, travel, and other chapters.

## Core principles

- The user's story belongs to the user.
- AI is a consultant, not the owner of the story.
- ZEKE preserves uncertainty when uncertainty exists.
- Planned is not completed.
- Scheduled is not taken.
- Historical context is not causation.
- Review before permanent save.
- Privacy is domain-specific.
- The interface is part of the conversation.
- ZEKE asks questions to improve understanding, not to maximize data collection.
- Respect the Story.

## First milestone

Build **ZEKE Prototype 1.0**, a working system that can be used daily with real data while preserving trust, transparency, and user control.
